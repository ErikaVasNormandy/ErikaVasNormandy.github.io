Part 2: Privilege Escalation

Our goal is to edit files like /etc/passwd or /etc/sudoers to gain super user privileges on the host

The easiest way to go about this is to duplicate the existing /etc/passwd, modify the copy to include a user with sudo permissions, and replace the existing /etc/passwd file with the copy

Something that hackingarticles' tutorial pointed out was that you can identify commands and files with

find / -perm -u=s -type f 2>/dev/null


Find command shown that wget and cp command has the SUID permissions. This could be possible for escalating root privilege.

Just going around and identifying things that correspond to the web server

ex. we found index.html (typical landing page) in /var/www/html) along with info.php

Honestly the hardest part for me at this point was finding a way to create a file that my www-data user could edit. The original tutorial seemed to gloss over this

Just trying to find a place where I can edit files

Looks like we found one in /var/www/html/wordpress. Now, don't just cp /etc/passwd [passwd copy] because it will still have the root permissions for editting that way.

.

You'll need to cat the contents of /etc/passwd and paste it into the file directory where you do have edit permissions

We need to generate a passwd entry for an example user called testuser, and use a text editor like vi to add this entry

cp [passd copy with username entry] /etc/passwd Verify the change with cat /etc/passwd

Right after copying over the editted /etc/passwd file, you can try out the testuser
